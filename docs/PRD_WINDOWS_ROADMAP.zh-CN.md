# VibeMouse Windows 版 PRD（产品需求文档 + 版本规划）

更新日期：2026-03-02  
文档状态：Draft v1（用于对齐方向，未开始对应功能开发）

---

## 1. 背景与问题

当前 VibeMouse 在 Windows 下已具备可用链路（侧键录音、转写、自动粘贴、OpenClaw 可切换），但仍有 3 个核心机会：

1. **产品化形态不足**：目前偏脚本/CLI，离“可分发应用”还有距离。  
2. **文本质量不足**：口语语气词（如“嗯、啊、那个”）会污染输出。  
3. **场景切换不足**：缺少“中文口述 -> 英文输出”的快捷翻译模式。
4. **输入触发冲突**：浏览器/系统会消费鼠标侧键默认前进后退行为。
5. **反馈时机滞后**：当前以“结束录音后统一输出”为主，缺少实时可见文本。

---

## 2. 产品目标

### 2.1 总目标
把 VibeMouse 从“工程可用工具”升级为“日常高频输入产品（Windows 优先）”。

### 2.2 阶段目标（按优先级）
1. **准确率优先模式**：默认提供“精度优先”配置（可接受略慢）。  
2. **应用化交付**：提供可双击启动、可视化状态、后续可打包 EXE。  
3. **文本后处理**：可配置去除语气词。  
4. **翻译模式**：快捷键切换“原文输出 / 中译英输出”。
5. **输入模式双轨**：保留鼠标侧键，同时支持用户自定义 Hotkey 模式。
6. **实时显示模式**：支持边说边出字（可切换）。

---

## 3. 目标用户与典型场景

### 3.1 目标用户
- 需要中英混合输入的开发者/创作者。  
- 长时间键盘输入，希望用语音替代高频文本输入。

### 3.2 典型场景
- 聊天/写文档：中文为主，夹英文术语。  
- 写代码注释/提交信息：中文口述后输出英文。  
- 连续高频输入：要求低延迟 + 少误触 + 可回退。

---

## 4. 产品范围

## 4.1 In Scope（本 PRD 覆盖）
- Windows 可视化启动器（已有最小 GUI，继续增强）。  
- “速度优先/精度优先”双模式。  
- 语气词清理开关（本地规则优先）。  
- 中译英模式快捷键切换。  
- 与 OpenClaw 模式共存（非强依赖）。
- 双输入模式：`mouse` + `hotkey`（用户可选）。
- 双输出时机：`end_of_utterance` + `streaming_preview`（用户可选）。

## 4.2 Out of Scope（暂不做）
- 云端账号体系/多设备同步。  
- 复杂多人协同/团队后台。  
- macOS 全量适配（后续单独规划）。

## 5. 功能需求（FR）

### FR-001 基础输入（已实现，需保持）
- 前侧键：开始/结束录音。  
- 后侧键空闲：Enter。  
- 后侧键录音中：按路由策略输出（普通或 OpenClaw）。  
- 任一失败有可见回退（剪贴板/日志提示）。

**验收标准**  
- 状态机行为不变。  
- 全量单测通过。

### FR-005 输入模式：Mouse / Hotkey 双轨
- 增加输入模式配置：`VIBEMOUSE_INPUT_MODE=mouse|hotkey`。  
- `mouse` 模式：保持现有前/后侧键状态机。  
- `hotkey` 模式：使用用户自定义复杂组合键触发（前键=开始/结束录音，后键=录音态结束并提交）。  
- 保持两模式语义一致，便于迁移。

**验收标准**  
- 可在设置中切换输入模式。  
- `hotkey` 模式可规避浏览器前进/后退冲突。  
- 两模式下核心状态机一致。

### FR-010 模式切换：速度优先 / 精度优先
- 提供用户可见模式开关：  
  - 速度优先：`funasr_onnx`  
  - 精度优先：`funasr`
- 默认模式改为“精度优先”（可配置）。

**验收标准**  
- 模式切换后重启生效。  
- 两模式均可完成中英混合转写。

### FR-020 语气词清理（可开关）
- 支持开启“语气词清理”。  
- 默认清理词库（首版）：`嗯/啊/呃/那个/就是/然后...`（可配置）。  
- 保留“不过度删除”原则：只删独立语气词，不删合法词干。

**验收标准**  
- 可在设置中开关。  
- 对常见口语文本有明显净化效果。  
- 误删率可控（见指标）。

### FR-030 中译英输出模式（快捷键切换）
- 增加“翻译模式”状态：  
  - OFF：输出原语言转写  
  - ON：中文口述输出英文
- 提供全局快捷键切换（如 `F9`，可配置）。

**验收标准**  
- 状态切换有 UI 提示。  
- ON 模式下输出英文；OFF 模式保持原文。  
- 不影响基础状态机与 OpenClaw 路由开关。

### FR-040 Windows 应用化交付
- 提供稳定 GUI 启动入口（已具备基础版）。  
- 提供 `.bat` 一键启动脚本（已具备）。  
- 后续提供 EXE 打包产物（PyInstaller）。

**验收标准**  
- 双击即可启动。  
- GUI 可显示运行状态/录音动态/日志。  
- 不依赖手工命令行操作。

### FR-050 实时显示模式（Streaming Preview）
- 新增输出模式配置：  
  - `end_of_utterance`：结束录音后一次性输出（当前模式）  
  - `streaming_preview`：录音中增量显示，结束时给最终稿  
- 提供可视状态：实时草稿与最终文本区分（防止误以为最终结果）。

**验收标准**  
- 用户可切换两种模式。  
- `streaming_preview` 模式下，口述过程中可见持续文本更新。  
- 结束录音后可得到最终修订文本。

## 6. 非功能需求（NFR）

- **NFR-性能**：常规 5~10 秒语音，端到端可感知延迟尽量 < 3 秒（精度模式允许上浮）。  
- **NFR-实时反馈**：`streaming_preview` 模式下，首个可见文本延迟目标 < 600ms。  
- **NFR-稳定性**：长时间运行不崩溃，异常有提示可恢复。  
- **NFR-可观测性**：日志可见，状态文件可读。  
- **NFR-隐私**：默认本地处理；涉及翻译引擎时必须明确“本地/云端”。

---

## 7. 指标与验收口径（建议）

1. **转写准确性（主观+样本）**  
   - 中英混说测试集相对当前版本错误率下降。  
2. **时延**  
   - 速度模式维持当前快速体验。  
   - 精度模式时延可接受（用户主观可用）。  
   - 实时模式首字延迟与稳定刷新频率达标。  
3. **语气词清理效果**  
   - 清理召回率高，误删可控。  
4. **可用性**  
   - 新用户 5 分钟内可启动并完成首次转写。

## 8. 版本规划（Roadmap）

## v0.2（应用可用版，近期）
- 稳定 Windows 启动器（GUI + bat）。  
- 增加“速度/精度”模式切换。  
- 固化默认配置与故障提示。  
- 增加 `hotkey` 输入模式（与 mouse 并存）。

**里程碑出口**：非开发用户可独立双击使用。

## v0.3（文本质量版）
- 上线语气词清理开关。  
- 支持自定义词库与黑白名单。  
- 增加清理前后对照日志（可选）。

**里程碑出口**：口语输入文本可直接用于文档/聊天，后编辑量明显下降。

## v0.4（中译英模式版）
- 快捷键切换翻译模式。  
- 集成翻译后处理策略（术语保护、大小写、标点）。  
- 增加模式状态可视反馈。
- 增加实时显示模式（streaming preview）可切换。

**里程碑出口**：中文口述可稳定输出英文草稿。

## v0.5（产品化发布版）
- EXE 打包与安装流程。  
- 开机启动、托盘驻留、自动更新（可选）。  
- 完整设置页。

## 9. 风险与对策

1. **准确率 vs 速度冲突**  
   - 对策：双模式并存，不强行单方案。  
2. **翻译质量波动**  
   - 对策：先做可切换与可回退，再逐步优化术语策略。  
3. **Windows 输入法/焦点兼容**  
   - 对策：保持剪贴板回退链路，增加可观测日志。  
4. **依赖复杂（Python/模型）**  
   - 对策：固定 Python 3.11，后续提供 EXE 降低环境成本。

---

## 10. 待确认项（和你对齐）

1. 精度模式是否直接设为默认。  
2. 语气词清理的默认词库（是否包含“就是/然后”这类高频词）。  
3. 中译英的引擎策略：纯本地优先，还是允许云端可选。  
4. GUI 视觉方向：极简状态窗 / 托盘风格 / 带波形动画。
5. Hotkey 默认推荐组合是否接受高复杂度方案（如四修饰键 + 功能键）。
6. 实时显示默认是否开启（建议默认关闭，按需开启）。
